# 1. functions/src/boutique/createOrder.js (CALLABLE)

CalyMob roept aan na cart-checkout. Ontvangt buyer + items + delivery preferences, doet:
1. Validate input (auth required, items non-empty, products bestaan in clubs/{clubId}/products)
2. Stock reservation in een transaction:
   - Voor elke item: lookup product variant
   - Als inventoryMode='tracked' EN stockCount<qty AND !allowBackorder: reject met error 'OUT_OF_STOCK'
   - Als inventoryMode='tracked' EN stockCount>=qty: reserveer (decrement stockCount in transaction)
   - Als inventoryMode='preorder' OF allowBackorder met stockCount=0: line.fulfillmentStatus='awaiting_restock'
3. Generate orderNumber: 'BTQ-2026-NNNN' via atomic counter clubs/{clubId}/settings/order_counter (zelfde pattern als ogmService)
4. Generate OGM via Cloud Functions admin SDK call naar OGM service (NIET zelf opnieuw bouwen — als de functions/ folder al een OGM utility heeft, hergebruik. Zo niet: laat een TODO en gebruik een placeholder zoals "TODO_OGM_GENERATE")
5. Compute totals: itemsSubtotal + deliverySurcharges + total
6. Create order doc clubs/{clubId}/orders/{orderId}:
   - status='awaiting_payment', payment.status='pending'
   - expiresAt = now + 72h
   - migration_source: null, _backfill: false (bewust expliciet, zodat triggers er rekening mee houden)
   - createdAt/updatedAt: serverTimestamp()
7. Return { success, orderId, orderNumber, ogm, ogm_display, total }

Errors te returnen:
- 'OUT_OF_STOCK' { productId, variantId, requested, available }
- 'PRODUCT_NOT_FOUND'
- 'PRODUCT_ARCHIVED'
- 'INVALID_INPUT'

# 6. functions/src/cotisations/createCotisationPayment.js (CALLABLE)

CalyMob roept dit aan vanuit "Renouveler ma cotisation":
- Auth required, request.auth.uid moet bestaan in clubs/{clubId}/members
- Bepaal categorie automatisch uit member.clubStatuten + huidige actieve saison
- Compute period: 'jan_dec' als now < 1 sept, anders 'sept_dec'
- Lookup tariff in clubs/{clubId}/membership_seasons (PLAN.md §13.1)
- Generate OGM met context_type='COTISATION', context_id=cotisationPaymentId
- Create cotisation_payment doc met status='awaiting_payment'
- Compute validity_until based on period
- Send QR email via bestaand sendPaymentQrEmail patroon
- Return { success, cotisationPaymentId, ogm, amount, period }

In MVP: skeleton met TODO voor de tariff-lookup en validity-computation. OGM gen + doc creation mag al echt zijn.
